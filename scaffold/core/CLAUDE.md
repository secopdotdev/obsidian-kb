# Knowledge Base — Agent Navigation Doctrine

> Scaffolded by `obsidian-kb` (`/kb-init`). This is your vault's navigation doctrine —
> edit it to taste. The toolkit's generator keeps the GENERATED files below in sync;
> everything else is hand-authored and stable.

## Entry protocol (cold-start)

1. Read `llms.txt` (root router — H1 + blockquote + H2 file lists; GENERATED, do not edit).
2. Read the relevant `_INDEX.md` MOC (grep-stable by numeric prefix; each is a routing TABLE).
3. Follow `[[wikilink]]` or `aliases` to the atomic note. Aliases resolve error codes,
   CLI invocations, and abbreviations to their canonical note.

> default) — never hand-edit cards/manifest/edges.

## Single-source-per-fact rule

| Fact class | Canonical home | Vault role |
|---|---|---|
| Project-internal (CLI ref, error table, config, architecture) | Each repo's `docs/` | GENERATED hub card links out — never copy |
| Cross-cutting (ADR metadata, atomic CLI/error, runbooks, ext catalog) | This vault | Canonical note (the graph node) |

Vault nodes mirroring repo content are **generated from harvested frontmatter** — they
cannot drift.

## Generated files — never hand-edit

Regenerated by the toolkit (`/kb-sync`); hand edits are overwritten on the next run:

- `llms.txt` — agent root router
- `00-meta/kb-manifest.json` — machine index
- `02-projects/<group>/**` — all project executive cards
- `03-adr/*.md`, `04-cli-errors/cmd-*.md` / `err-*.md`, `08-blockers/blk-*.md` — atomic notes
- Routing-table rows in `*/_INDEX.md` files (rows regenerated between `<!-- KB-SYNC:ROWS -->` markers)

Hand-authored stable files: this `CLAUDE.md`, `01-groups/*.md` MOCs, `00-meta/conventions.md`,
`00-meta/tag-taxonomy.md`, `_templates/`, `05-runbooks/*.md`, `07-toolkit/**`.

## Frontmatter is the API

All properties are YAML frontmatter (never inline `key:: value`). The schema lives in
`00-meta/conventions.md`. Bases/Dataview queries and agent parsing both run on frontmatter.

## Wikilink resolution

`[[note-title]]` resolves via shortest-path. `aliases:` in frontmatter lets error codes
(`exit-2`), CLI invocations, and abbreviations resolve to the canonical note. Always set
aliases on atomic notes.
